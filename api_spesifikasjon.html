Utbetalinger

API for iverksetting av utbetalinger mot Oppdragssystemet

Hent utbetaling

GET
/utbetalinger/{uid}

Henter utbetalingen slik den ser ut hos oss.

Path-parametre

  • uid

    Påkrevd

    string (uuid) 
    Eksempel: 7832faed-2140-4884-9e5c-5ab434e57fbb

    ID til utbetalingen. Sendes ikke til OS/UR.

Respons

400: Ugyldig format på uid-parameter
404: Fant ikke utbetalingen

Send utbetaling

POST
/utbetalinger/{uid}

Oppretter en utbetaling hos oss som vi effektuerer i Oppdragssystemet.

Path-parametre

  • uid

    Påkrevd

    string (uuid) 
    Eksempel: 7832faed-2140-4884-9e5c-5ab434e57fbb

    ID til utbetalingen. Sendes ikke til OS/UR.

Request body:

application/json

Påkrevd

  • sakId

    Påkrevd

    string [1 ... 25] tegn
    Eksempel: ASD123NLDQ23

    Id på saken i vedtaksløsningen.

  • behandlingId

    Påkrevd

    string [1 ... 30] tegn
    Eksempel: ADNLK123lDLJNWQ1

    Id på behandlingen i vedtaksløsningen. Brukes for å spore utbetalingen tilbake til den konkrete hendelsen som trigget den. Kan også være id på vedtak, meldekort etc. DVH bruker denne når de sammenstiller stønadsstatistikk.

  • personident

    Påkrevd

    Eksempel: 15507600333

    Fødselsnummer eller D-nummer

  • vedtakstidspunkt

    Påkrevd

    Eksempel: 2024-11-26T09:34:10.912Z

    Tidspunkt vedtaket ble fattet. Bruker ISO 8601-format.

  • stønad

    Påkrevd

    string 
    Eksempel: ARBEIDSSØKER_ORDINÆR

    Stønadstypen som gjelder for utbetalingen

    En av:

    • PERMITTERING_ORDINÆR
    • PERMITTERING_FISKEINDUSTRI
    • EØS
      • ...
  • beslutterId

    Påkrevd

    string 
    Eksempel: Z123456

    Enten NAV-ident på beslutter når behandlingen er gjort manuelt, eller navn på system/applikasjon/servicebruker dersom behandlingen er gjort automatisk. Utsjekk gjør ingen validering av dette feltet.

  • saksbehandlerId

    Påkrevd

    string 
    Eksempel: A234567

    Enten NAV-ident på saksbehandler når behandlingen er gjort manuelt, eller navn på system/applikasjon/servicebruker dersom behandlingen er gjort automatisk. Utsjekk gjør ingen validering av dette feltet.

  • periodeType

    Påkrevd

    string 
    Eksempel: UKEDAG

    Type periode periodene skal tolkes som. F.eks. 'UKEDAG' er vanlige ukedager (man-fre) mens 'DAG' er alle dager (man-søn). 'MDN' er hele måneder og 'EN_GANG' er engangsbeløp som utbetales med en gang.

    En av:

    • UKEDAG
    • DAG
    • MND
    • EN_GANG

Endre utbetaling

PUT
/utbetalinger/{uid}

Endrer en eksisterende utbetaling. Brukes typisk om man skal foreta en revurdering/korrigering av en tidligere utbetalt periode. De nye periodene man sender inn må ha samme periodetype som de gamle. Hvis man f.eks. tidligere har sendt inn perioder med periodetype 'MND' kan man ikke her sende inn nye perioder med periodetype 'DAG'. Om man av en eller annen grunn trenger å endre periodetype må man annullere utbetalingen.

Annullér utbetaling

DELETE
/utbetalinger/{uid}

Sletter utbetalingen hos oss og annullerer utbetalingen i Oppdragssystemet. Dette vil typisk opprette en tilbakekrevingssak. For å unngå at man sletter feil utbetaling ved et uhell (f.eks. hvis man har feil/utdatert utbetaling), må man sende inn hele utbetalingen man ønsker å slette som vi da validerer mot det vi har lagret hos oss.
